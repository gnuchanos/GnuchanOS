4. Dinamik Bellek — Tam Kılavuz

4.1 calloc — Sıfırlanmış Bellek

int* arr = (int*)calloc(10, sizeof(int)); // arr[0]..arr[9] = 0 (calloc sıfırlar) // malloc(10 * sizeof(int)) ile farkı: malloc sıfırlamaz

4.2 realloc — Yeniden Boyutlandırma

int* arr = malloc(5 * sizeof(int)); // kullan... // 10 elemana genişlet (eski veri korunur) int* temp = realloc(arr, 10 * sizeof(int)); if (temp == NULL) { // realloc başarısız! arr hala geçerli, free'le free(arr); return 1; } arr = temp; // yeni pointer // Küçültme: fazla kısım kaybolur arr = realloc(arr, 3 * sizeof(int));
⚠️ realloc önemli: Her zaman geçici pointer kullan! arr = realloc(arr, ...) başarısız olursa orijinal pointer kaybolur (memory leak).

4.3 Sık Yapılan Bellek Hataları

  • ❌ Memory leak: malloc/calloc/realloc ile alınan bellek free edilmezse program kapanana kadar tutulur
  • ❌ Double free: free(ptr); free(ptr); → heap corruption
  • ❌ Use-after-free: free(ptr); *ptr = 5; → undefined behavior
  • ❌ Buffer overflow: malloc(10) ile 20 eleman yazmak → heap corruption
  • ❌ Uninitialized read: malloc ile alınmış belleği okumak (calloc kullan)
  • ❌ realloc NULL kontrolü yapmamak: veri kaybı

5. Bellek Haritası (Memory Layout)

// Bir C programının bellek haritası (adres yukarıdan aşağı): // ┌─────────────────────┐ yüksek adres // │ Stack │ local değişkenler, fonksiyon çağrıları // │ ↓ │ // │ ↑ │ // │ Heap │ malloc/calloc/realloc ile alınan bellek // ├─────────────────────┤ // │ BSS (Block) │ başlatılmamış global/static değişkenler (0) // │ Data Segment │ başlatılmış global/static değişkenler // │ Text (Code) │ program kodu (read-only) // └─────────────────────┘ düşük adres int global_init = 5; // Data Segment static int global_uninit; // BSS (0) int main() { int local = 10; // Stack int* heap = malloc(100); // Heap free(heap); return 0; }
💡 Stack overflow: çok derin recursion veya çok büyük local dizi → stack taşar. malloc kullan.
