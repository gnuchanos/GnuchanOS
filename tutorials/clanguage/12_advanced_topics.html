Lesson 12: Advanced C Topics — Detaylı Referans

1. Variadic Functions — Değişken Argümanlar

Fonksiyonların değişken sayıda argüman almasını sağlar. gerekli.

#include #include // ... ile değişken argüman double average(int count, ...) { va_list args; va_start(args, count); // başlat double sum = 0; for (int i = 0; i < count; i++) sum += va_arg(args, double); // sıradaki double va_end(args); // temizlik return sum / count; } int main() { printf("%.2f\n", average(3, 10.0, 20.0, 30.0)); // 20.0 return 0; }

1.1 Kendi printf'inizi Yazın

void my_printf(const char* fmt, ...) { va_list args; va_start(args, fmt); vprintf(fmt, args); // vprintf ile doğrudan yazdır va_end(args); }

2. setjmp / longjmp — Sıçrama

setjmp/longjmp ile fonksiyonlar arası atlama yapılabilir. Exception handling benzeri bir mekanizma.

#include #include jmp_buf buf; void error_handler() { printf("Hata! Başa dönülüyor...\n"); longjmp(buf, 1); // setjmp'nin olduğu yere dön, 1 döndür } int main() { if (setjmp(buf) == 0) { // ilk çağrı: 0 döner printf("Normal akış\n"); error_handler(); printf("Bu satır çalışmaz\n"); } else { // longjmp sonrası: 1 döndü printf("Kurtarıldı!\n"); } return 0; }
⚠️ longjmp ile atlandığında local değişkenlerin değeri tanımsız olabilir (volatile kullanın).

3. volatile Keyword

volatile derleyiciye "bu değişkeni optimize etme, her seferinde bellekten oku" der. Donanım register'ları, sinyal işleyicileri, çok thread'li değişkenler için kullanılır.

volatile int flag = 0; // derleyici her okumada belleğe gider void wait_for_flag() { while (!flag) { // volatile olmazsa derleyici flag'i register'a alır, sonsuz döngü! // bekle } }

4. _Generic (C11) — Tür Seçimi

_Generic ile tür bazında farklı kod seçimi yapılabilir. Overloading benzeri.

#include #define TYPE_NAME(x) _Generic((x), \ int: "int", \ float: "float", \ double: "double", \ char*: "string", \ default: "diğer" \ ) int main() { printf("%s\n", TYPE_NAME(42)); // int printf("%s\n", TYPE_NAME(3.14f)); // float printf("%s\n", TYPE_NAME("merhaba")); // string return 0; }

5. Alignment — _Alignas, _Alignof (C11)

Bellek hizalamasını kontrol eder. Bazı veri türleri belirli adreslerde olmalıdır (ör. int → 4'ün katı).

#include #include // 16-byte hizalı buffer alignas(16) char buffer[64]; // SIMD işlemleri için 16-byte hizalama gerekebilir printf("Alignment of double: %zu\n", alignof(double)); // 8

6. Signal Handling — sinyal.h

Programa gönderilen sinyalleri yakalamak için (Ctrl+C, segmentation fault, vs.)

#include #include void handle_sigint(int sig) { printf("\nCtrl+C yakalandı! (sinyal %d)\n", sig); // cleanup yap } int main() { signal(SIGINT, handle_sigint); // Ctrl+C yakala printf("Ctrl+C'ye basmayı dene...\n"); while(1); // sonsuz döngü return 0; }

7. assert.h — Debug Assertions

#include int divide(int a, int b) { assert(b != 0); // b=0 ise program durur, hata gösterilir return a / b; } // NDEBUG tanımlıysa assert'ler devre dışı: // #define NDEBUG

8. Compile-Time İşlemler — static_assert (C11)

#include // Derleme zamanında kontrol static_assert(sizeof(int) == 4, "int 4 byte olmalı!"); static_assert(10 > 5, "Bu her zaman doğru"); // Eğer koşul yanlışsa → derleme hatası!

9. Flex Array Member (C99)

Struct'ın sonunda boyutsuz dizi tanımlamak için kullanılır. Dinamik boyutlu struct.

typedef struct { int len; int data[]; // flexible array member (son alan olmalı) } FlexArray; FlexArray* fa = malloc(sizeof(FlexArray) + 10 * sizeof(int)); fa->len = 10; fa->data[0] = 42; free(fa);
