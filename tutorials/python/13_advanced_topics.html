Decorators

A decorator is a function that wraps another function to extend its behavior without changing its code. The syntax is @decorator_name above the function.

Basic Decorator

def my_decorator(func): def wrapper(): print("Something before the function") func() print("Something after the function") return wrapper @my_decorator def say_hello(): print("Hello!") say_hello()
Something before the function Hello! Something after the function

@timer Decorator

import time def timer(func): def wrapper(*args, **kwargs): start = time.time() result = func(*args, **kwargs) end = time.time() print(f"func took {end - start:.4f} seconds") return result return wrapper @timer def slow_add(a, b): time.sleep(0.5) return a + b print(slow_add(3, 4))
func took 0.5001 seconds 7

@log Decorator & functools.wraps

import functools def log(func): @functools.wraps(func) def wrapper(*args, **kwargs): print(f"Calling {func.__name__}") result = func(*args, **kwargs) print(f"Returned: {result}") return result return wrapper @log def multiply(x, y): """Multiply two numbers.""" return x * y print(multiply.__name__) # multiply (not wrapper!) print(multiply(4, 5))
multiply Calling multiply Returned: 20 20
💡 functools.wraps preserves the original function's metadata (name, docstring, etc.) so the wrapper looks like the original function.

Decorator with Arguments

def repeat(n): def decorator(func): def wrapper(*args, **kwargs): for _ in range(n): func(*args, **kwargs) return wrapper return decorator @repeat(3) def say_hi(): print("Hi there!") say_hi()
Hi there! Hi there! Hi there!

Generators

A generator yields values one at a time, pausing between each yield. This is memory-efficient for large sequences.

Simple Generator

def count_down(n): while n > 0: yield n n -= 1 for num in count_down(5): print(num)
5 4 3 2 1

Generator Expressions

squares = (x ** 2 for x in range(10)) print(list(squares)) # Sum of squares total = sum(x ** 2 for x in range(10)) print(total)
[0, 1, 4, 9, 16, 25, 36, 49, 64, 81] 285

yield from

def chain(*iters): for it in iters: yield from it result = list(chain([1, 2], (3, 4), "ab")) print(result)
[1, 2, 3, 4, 'a', 'b']
💡 Generators only compute values when you iterate over them — they do not store all values in memory at once.

Context Managers

The with statement is used for resource management (files, locks, connections). You can create your own context managers.

Custom Context Manager (Class-based)

class ManagedFile: def __init__(self, filename): self.filename = filename def __enter__(self): print("Opening file...") self.file = open(self.filename, "w") return self.file def __exit__(self, exc_type, exc_val, exc_tb): print("Closing file...") self.file.close() with ManagedFile("test.txt") as f: f.write("Hello, context manager!") print("Done.")
Opening file... Closing file... Done.

Using contextlib.contextmanager

from contextlib import contextmanager @contextmanager def managed_file(filename): print("Opening...") f = open(filename, "w") try: yield f finally: print("Closing...") f.close() with managed_file("hello.txt") as f: f.write("Hello from contextmanager!") print("Done.")
Opening... Closing... Done.

Threading

Threading lets you run multiple tasks concurrently. Python threads are great for I/O-bound tasks.

Basic Thread

import threading import time def worker(name, delay): print(f"{name} started") time.sleep(delay) print(f"{name} finished") t1 = threading.Thread(target=worker, args=("A", 2)) t2 = threading.Thread(target=worker, args=("B", 1)) t1.start() t2.start() t1.join() t2.join() print("All threads done.")
A started B started B finished A finished All threads done.

Daemon Threads

t = threading.Thread(target=lambda: time.sleep(5)) t.daemon = True t.start() print("Main exits — daemon dies automatically.")
Main exits — daemon dies automatically.
⚠️ Daemon threads are killed abruptly when the main process ends. Use join() if you need them to finish.

Threading.Lock

lock = threading.Lock() counter = 0 def increment(n): global counter for _ in range(n): with lock: counter += 1 threads = [threading.Thread(target=increment, args=(100000,)) for _ in range(4)] for t in threads: t.start() for t in threads: t.join() print(f"Counter: {counter}")
Counter: 400000

Thread Pool (concurrent.futures)

from concurrent.futures import ThreadPoolExecutor def square(x): return x * x with ThreadPoolExecutor(max_workers=3) as executor: results = list(executor.map(square, [1, 2, 3, 4, 5])) print(results)
[1, 4, 9, 16, 25]

Comprehensions Recap

List Comprehension

squares = [x ** 2 for x in range(10) if x % 2 == 0] print(squares)
[0, 4, 16, 36, 64]

Dict Comprehension

square_dict = {x: x ** 2 for x in range(5)} print(square_dict)
{0: 0, 1: 1, 2: 4, 3: 9, 4: 16}

Set Comprehension

even_set = {x for x in range(10) if x % 2 == 0} print(even_set)
{0, 2, 4, 6, 8}

Nested Comprehension

matrix = [[i * j for j in range(1, 4)] for i in range(1, 4)] print(matrix)
[[1, 2, 3], [2, 4, 6], [3, 6, 9]]

Walrus Operator

The walrus operator := (assignment expression) lets you assign a value and use it in the same expression.

# Without walrus n = int(input("Enter a number: ")) if n > 0: print(f"Positive: {n}") # With walrus if (n := int(input("Enter a number: "))) > 0: print(f"Positive: {n}") # Reading a file line-by-line with open("test.txt", "r") as f: while (line := f.readline()): print(line.strip())
Positive: 7 Positive: 7 Hello, context manager! Hello from contextmanager!
💡 Use parentheses around the walrus expression: if (x := expr) > 0: to make it clear.
