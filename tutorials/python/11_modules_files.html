Lesson 11: Modules and File I/O

Simple English — Based on W3Schools Python Tutorial

Importing a Module

A module is a file containing Python code (functions, classes, variables) that you can reuse. Use import to bring in a module.

import math print(math.pi) print(math.sqrt(16)) print(math.ceil(3.7))
3.141592653589793 4.0 4

from ... import

You can import specific items from a module instead of the whole module.

from math import pi, sqrt print(pi) print(sqrt(25))
3.141592653589793 5.0

Alias (as)

Use as to give a module or function a shorter name.

import math as m from datetime import datetime as dt print(m.floor(4.9)) print(dt.now())
4 2025-01-15 14:30:00.123456

dir() - List Module Contents

The dir() function returns all the names inside a module.

import math print(dir(math)[:10]) # show first 10 items
['__doc__', '__loader__', '__name__', '__package__', '__spec__', 'acos', 'acosh', 'asin', 'asinh', 'atan']

pip install - Installing External Packages

pip is Python's package manager. You install third-party modules from the internet.

# In terminal / command prompt: # pip install requests # pip install numpy # Then in your code: print("Use 'pip install packagename' to install packages") print("Then import them in your code!")
Use 'pip install packagename' to install packages Then import them in your code!

Creating Your Own Module

Any .py file is a module. You can create one and import it from another script.

# Save this as mymodule.py def hello(name): return f"Hello, {name}!" PI = 3.14159 # In another file: # import mymodule # print(mymodule.hello("Alice")) # print(mymodule.PI) print("Module created! You can import it.")
Module created! You can import it.

The __name__ == "__main__" Idiom

This checks if the script is run directly (not imported). Useful for test code.

# mymodule.py def add(a, b): return a + b if __name__ == "__main__": # This runs only when the file is executed directly print("Running tests...") print(add(2, 3))
Running tests... 5

File I/O - open() Function

Use open() to work with files. Common modes:

ModeMeaning
"r"Read (default)
"w"Write (overwrites existing file)
"a"Append (adds to end)
"x"Exclusive creation (fails if file exists)
"r+"Read and write
"b"Binary mode (add to another mode, e.g. "rb")
# Writing to a file file = open("test.txt", "w") file.write("Hello, File!\n") file.write("Second line.") file.close() # Reading from a file file = open("test.txt", "r") content = file.read() file.close() print(content)
Hello, File! Second line.

Reading Methods: read(), readline(), readlines()

# Assuming test.txt contains 3 lines file = open("test.txt", "r") # readline() reads one line at a time line = file.readline() print("First line:", line) file.seek(0) # go back to start # readlines() returns a list of lines lines = file.readlines() file.close() print("All lines:", lines)
First line: Hello, File! All lines: ['Hello, File!\n', 'Second line.']

Writing to a File

# Write numbers 1 to 5 file = open("numbers.txt", "w") for i in range(1, 6): file.write(f"Number {i}\n") file.close() # Read and print file = open("numbers.txt", "r") print(file.read()) file.close()
Number 1 Number 2 Number 3 Number 4 Number 5

With Statement - Context Manager

The with statement automatically closes the file when the block ends. It is the recommended way to handle files.

with open("test.txt", "r") as file: content = file.read() print(content) # No need to call close() - it's automatic!
Hello, File! Second line.

Working with JSON

JSON is a lightweight data format. Python's json module converts between JSON strings and Python objects.

import json # Python dict - JSON string data = {"name": "Alice", "age": 30, "city": "London"} json_str = json.dumps(data, indent=2) print(json_str) # JSON string - Python dict parsed = json.loads(json_str) print(parsed["name"])
{ "name": "Alice", "age": 30, "city": "London" } Alice
# Write JSON to a file import json data = {"score": 95, "level": 3} with open("data.json", "w") as f: json.dump(data, f) # Read JSON from a file with open("data.json", "r") as f: loaded = json.load(f) print(loaded)
{'score': 95, 'level': 3}

CSV Module

The csv module handles comma-separated value files.

import csv # Writing CSV with open("people.csv", "w", newline="") as f: writer = csv.writer(f) writer.writerow(["Name", "Age"]) writer.writerow(["Alice", "30"]) writer.writerow(["Bob", "25"]) # Reading CSV with open("people.csv", "r") as f: reader = csv.reader(f) for row in reader: print(row)
['Name', 'Age'] ['Alice', '30'] ['Bob', '25']

try/except/finally with Files

Always handle errors when working with files, especially if the file might not exist.

try: with open("missing.txt", "r") as f: content = f.read() except FileNotFoundError: print("File not found!") finally: print("This always runs.")
File not found! This always runs.

Pathlib - Modern File Paths

The pathlib module provides an object-oriented way to handle file paths.

from pathlib import Path # Create a Path object path = Path("test.txt") print(path.exists()) # Does the file exist? print(path.name) # File name print(path.stem) # Name without extension print(path.suffix) # Extension (.txt) print(path.read_text()) # Read entire file
True test.txt test .txt Hello, File! Second line.
Summary: Modules let you reuse code from other files. pip installs third-party packages. File I/O with open() and with lets you read and write data to disk. JSON and CSV are standard formats for data exchange. The pathlib module is the modern Python way to work with paths.
